# 读者来信

- 原文：[We Get Letters](https://freebsdfoundation.org/wp-content/uploads/2023/01/letters.pdf)
- 作者：**Michael W Lucas**

亲爱的最后一线希望：

实在太多了。我把服务器清理好了，故障处理了，环境也稳定下来了，但老板还不停地堆活儿，一件接一件，根本做不完。我又不能辞职，这可怎么办？

——焦头烂额的人

亲爱的焦头烂额：

要帮你，先得改写你的信，把责任归到该归的地方。

"我以最低限度的胜任履行了职责，但我不理解，干得好得到的奖赏就是更多的活儿。我向经理说明后续任务所需工作量的努力失败了，要么是我的沟通能力不足，要么是经理真心不认为工作量有什么相关，因为他是个正冲向 C 级高管的反社会者。多半是后者。我把生活安排得既无远见又无弹性，所以现在被困住了。我该怎么办？"

这样才对。

计算机专业人士的问题在于，他们把自己看作"问题解决者"。毕竟，他们让那些超级复杂的机器做出复杂的事情，比如把两个很大的数加起来。每个处理器核心里有十亿百亿千亿兆个晶体管，你统帅它们全部！当这台复杂得可怕的机器崩溃时，你来修！你不"和"软件打交道——你是职业问题解决者。

问题解决者。这正是那个正冲向 C 级高管的反社会者想让你认为的。

计算机是简单的。最先进的计算机，其复杂程度也远远不足以理解最简单的病毒的精妙，而病毒在企业的政治恐怖面前又黯然失色。计算机专业人士擅长在严格约束的环境中解决问题——那里由四大神圣资源（处理器、I/O、存储、内存）主宰，但这些技能无法模拟现实世界中数不清的资源。

要在现实世界立足，你必须接受自己技能的局限，并抛弃"问题解决者"这一身份。只要你还以这种自我形象自居，就会输给现实世界的非逻辑。你有指标，你有度量，你有所有数据证明自己工作努力。抛弃别人强加给你的标签，解放自己。放弃解决问题，转而选择战略性失败。

战略性失败不是把整个系统搞垮。任何系统管理员都能做到这一点。它甚至不在于时机，尽管时机很重要。它在于选择合适的失败，在合适的时间让合适的人难堪，并能面不改色地声明："我只有时间维护一套系统，我选了关键的那套。"

是的，你的经理会生气。那又怎样？你不能辞职，他们也换不掉你。处于你这样位置的人，在任何组织里，都拥有从古怪到不堪的独一无二的技能组合，这种混合在任何其他人身上都完全无法复制。培养这些技能的唯一办法就是做你自己，没人会自愿承担。别傲慢——毕竟，在计算这个小小的专业领域之外，任何称职的人会让自己被算计到这种境地吗？表现愤怒和沮丧只会让你被送去人力资源部接受"态度"辅导。耸耸肩，淡淡地说一句"我按照管理层的指示分配了我的资源"——这话对你大有帮助。使用这些字眼，"按照管理层的指示"，就像烤大蒜一样。这里撒一点，那里撒一点，会让你公事公办的态度显得可信。

是的，你的经理会生气。那又怎样？

经过两三次精心选择、高可见度、令人恼火但非毁灭性的战略性失败之后，你会和经理以及形形色色的局外人开会，他们想知道你为什么这么糟糕。你的经理巴不得把你从最近的气闸扔出去，所以别理他。专注于其他人。保持冷静。向除经理外的所有人展示你工作方式的文档。通常，仅仅有文档就足够了。局外人不会问太多问题，出于一种合理的恐惧——他们怕自己会学到计算机知识，从而被强行调到你的部门。

人们会提出解决方案。你也应该提供方案。其中一个是你的偏好，其余的可接受。如果公司让你，比如说，停止聚合 syslog 和 netflow 数据并关闭这些系统，那也行。你完全可以这样回复工单："管理层已声明，我不能帮你解决这个问题。"

最终，他们会决定再招一个人。如前所述，他们招的人既不胜任帮你，也不胜任管理系统。记住系统管理员守则第 27 条："胜任的同事不是招来的，是炼出来的。由你来炼。"如果你主动提出带教一名初级系统管理员，为公司省下数万美元，你获得帮助的几率就会提高。如果你在他人面前提出这个建议，你就能改善自己在公司的形象。一定要说，你对申请人有具体的提问。

跟求职者交谈？呃。我知道，很痛苦，但你以后每天都要和幸存者说话，所以最好把那些难以合作的人筛掉。别问二叉树或冒泡排序或任何这类垃圾。你要尽快筛掉申请人，所以设计一道谜题。既然技术已经进步，CRT 显示器不再是标配，我终于可以分享我的秘密服务台招聘谜题了。

我会把申请人带到一间孤立的房间，用整栋楼里最糟糕的荧光灯管照明。如果有机会提前准备，我会放上 CD《伟大的恐怖电影尖叫》，音量调到几乎听不见，再点上一支"绝望系列"的熏香样品，营造气氛。房间里有桌子、电脑和一台画面扭曲变形的 CRT 显示器。我会说："如果你为我工作，你会怎么修这个？给我讲讲思路。"

很简单，对吧？

换显示器不管用。换显卡或整台电脑也不管用。到了这一步，大多数申请人说他们会把整台电脑寄回厂家。

那些意识到问题在于电脑的位置，把它从我用胶带粘在桌底的磁铁旁移开两英尺的人？他们得到了工作。

你需要一道类似的谜题，用现代技术，再加一丝恶意。最好连你的老板都能理解。"问题解决者"标签的便利之处在于，计算部门以外的人也相信它。"我设了一道我必须解决的典型问题，只有这些申请人能解出来"——这种说法立刻可信。

是的，你要花时间培训新的小弟——但干得好得到的奖赏就是更多活儿。另外，你培训他们是为了处理你不愿干的活儿，而你已经知道他们的解题技能足以应对严格约束的计算机环境。这会给你腾出时间，去解决你真正的问题，重新安排生活，让它更有弹性。

不过，使用战略性失败要小心。用得太多，你会发现自己被火箭送上了 C 级。
