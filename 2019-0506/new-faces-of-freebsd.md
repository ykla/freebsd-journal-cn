# FreeBSD 新面孔

作者：**DRU LAVIGNE**

本专栏旨在聚焦近期获得提交权限的贡献者，并把他们介绍给 FreeBSD 社区。本期聚焦 4 月获得 src 权限的 Doug Moore，以及 5 月获得 docs 权限的 Sergio Carlavilla。

请介绍一下你自己、背景和兴趣。

- **Doug**：我从小在得州长大，算个聪明孩子，除了 Rice University 别无所求，之后也一直在它周围转悠。我在别处拿了计算机科学博士，回到 Rice 在学术生涯的下沿潜伏。后来一些 Rice 的人创立了一家初创公司，我跳了进去，两次收购之后我进了 Cisco 工作。那之后尘埃落定，我又回到 Rice 潜伏。

  我对离散数学问题的快速计算很感兴趣。我写过沿多维 Hilbert 曲线对点排序的代码，也写过求曲线上给定点之后进入某个盒子的下一点的代码。我觉得这是管理和搜索多维数据的好方法。我还写过在固定大小版本的 Gosper 持续对数上做算术的代码，比浮点好得多，因为它允许你表示 1/3 这样的概念。不过这些暂时还只是私人消遣，世人尚未买账。

- **Sergio**：我叫 Sergio Carlavilla，在西班牙瓦伦西亚（“Fallas”的故乡）出生并长大。

  目前在 ViewNext/IBM 担任全栈开发者（Angular/Java）。业余时间我喜欢用 C/C++ 编程，也是 QtEmu 的现任维护者。

  离开电脑时，我喜欢散步、读历史书、做饭。我真心热爱烹饪。我也喜欢在欧洲旅行，因为我觉得这是一片历史悠久、风景壮丽的大陆。

  我对 FreeBSD 的兴趣在于文本处理工具，例如我们的 `bsdgrep`、`cat` 等。

你是怎么知道 FreeBSD 的？FreeBSD 哪些方面让你感兴趣？

- **Doug**：Alan Cox 成为 Rice 教授时我认识了他，他是 FreeBSD 的长期提交者。他创立了我后来加入的公司，以 FreeBSD 为基础做 Web 缓存软件产品。我跟着他参加过几次 FreeBSD 聚会，希望自己也能在周围人谈论 FreeBSD 时有点事做。我想帮忙，但似乎做什么都需要我不具备的专门知识。

- **Sergio**：我第一次听说 FreeBSD 是在上网络课程学 TCP/IP 协议时。教授解释说该协议的最佳实现就在 FreeBSD 操作系统中。后来我看到其他操作系统都从 FreeBSD 借鉴想法，因为它文档优秀且稳健。最吸引我的是它优秀的文档，以及它作为完整操作系统发布这一点。所有东西同时构建，不是拼图。

你是怎么成为提交者的？

- **Doug**：Alan 指出一段他正在做的 FreeBSD 代码，问有没有什么想法。我觉得凭我所知的一点 C 语言技巧和几个数据结构，能写出比那里更好的代码，于是我写了。重复几次后，Konstantin Belousov 建议我成为提交者，这样我就不用一直缠着他和 Mark Johnston 提交改动了。现在改成缠着他们做代码评审了。

- **Sergio**：嗯，正如我之前所说，我对文档很感兴趣。作为程序员，使用操作系统、工具或框架时，我希望它有良好的文档，并希望理解它如何工作。

  我开始把 FreeBSD 翻译成我的母语西班牙语。我先翻译文章，然后把翻译发给 @gabor 校验并提交到 FreeBSD 仓库。接下来的几个月，@gabor 提名我获得提交权限，这样我就能自己执行提交了。

加入 FreeBSD Project 以来你的体验如何？对也想成为 FreeBSD 提交者的读者有什么建议？

- **Doug**：我搞砸了第一次大提交，接着又搞砸了修复。如果你的意图是好的，并且让其他人了解情况，大家似乎相当宽容。一个错误加一团谜才会让人恼火。新人只需耐心；你的导师对你来说可能是拦路虎，但对你导师而言你只是他十几项与 FreeBSD 相关的任务之一，所以要有耐心。

- **Sergio**：加入 FreeBSD Project 以来我的体验非常积极。社区的接纳非常棒。我们目前正尝试一种翻译 FreeBSD 文档的新方式，敬请期待。

  对想参与 FreeBSD 的人，我有这些建议：

  - 选你认为能改进的领域（src/docs/ports）并做出必要的修改。
  - 多和社区交流。幸运的是，FreeBSD 社区非常开放，乐于助人。
  - 要有耐心。如果开发者回复慢，不要绝望；FreeBSD Project 由志愿者组成。
  - 最后，应该乐在其中。

---

**DRU LAVIGNE** 是 FreeBSD 文档提交者，《BSD Hacks》和《The Best of FreeBSD Basics》的作者。
