# 会议报告

## COSCUP

作者：Benedict Reuschling

COSCUP（<https://coscup.org/2019/en>）是台湾台北的开源会议，自 2006 年起每年举办。今年的会议于 2019 年 8 月 17-18 日在台湾科技大学（NTUST）校园举行。我未能参加 2017 年的 BSD TW 会议（<https://2017.bsdtw.org/>），但人们一直告诉我他们在那里度过了一段美好时光，于是我决定向今年的 COSCUP 会议提交演讲。类似于 FOSDEM BSD 开发室——主会议中关于特定主题的独立分会场——COSCUP 有一个名为“BSDTW x Cat System Workshop”的 BSD 分会场。幸运的是，我的演讲被接受了，于是我预订了前往台湾的旅行和酒店。

周三下午到达后，入住酒店，我与 Li-Wen（他的朋友）以及从日本过来的 Hiroki Sato 一起吃了晚饭。这不仅是品尝当地美食的绝佳机会，也让我们能了解 FreeBSD 的最新动态，并一起规划接下来几天的安排。Li-Wen 多年来以不同身份参与该会议，因此认识很多人，并组织了那里的 BSD 专场。此外，FreeBSD 基金会申请了一个展台，用于与人交流和分发宣传材料，如传单、贴纸和笔。幸运的是，我们有足够的人手来值守展台，包括 Philip“trouble”Paeps、Sato-San、Li-Wen 和我。

Philip 在第一个正式会议日的前一天加入了我们。他带了一个装满额外纪念品的手提箱，他带着这个手提箱在该地区各个活动之间巡回推广 FreeBSD。他还将在周日的 BSD 专场做一场关于 ZFS 的演讲。我和 Hiroki Sato、Philip 一起去了一个巨大的地下地铁商场吃晚饭。我们聊了项目中的各种事情，以及从不同角度看待语言学习的趣事。

那天晚上，会议演讲者和值守展台的人被邀请到一个四楼露台上参加欢迎派对，那里俯瞰城市，离著名的台北 101 大楼不远。类似于 BSDCan 注册时的酒吧，人们可以在第一个会议日的前一天领取胸牌，同时与其他演讲者一起享用饮品和小吃。

每个人都拿到了自己的姓名胸牌，Sato-San 和 Philip 很快就由注册工作人员在他们的胸牌上写上了正式的中文字符。Philip 的昵称”trouble”对看到的人来说特别有趣，事实证明这是一个很好的话题开场白。我们遇到的每个新人都问他为什么叫 trouble。显然，这些字符有双重含义，但无论如何，我们不会孤单太久。

我自己的名字音译起来有点困难，但注册处的人用拼音字母为我的名字注音，所以我的胸牌上也有一些漂亮的中文字符。与一些当地人（FreeBSD Ports 维护者和翻译者）交谈后，再次见到 Li-Wen（他在会议前夕忙于一些组织工作），我们出发去登台北 101 大楼。顾名思义，这是一栋 101 层的建筑，是城里最高的，是一个很好的地标。排着长队等电梯带我们到上层，从那里我们看到了夜晚灯火辉煌的城市。风阻尼器也证明是一个有趣的技术亮点，因为那里播放的一些视频展示了这个巨大的配重在台风和强风中摇摆。

在第一个会议日，我乘坐当地的 MRT 地铁网络到达 NTUST，穿过一个公园步行到校园。在会议前几周，我以二维码的形式收到了免费的演讲者入场证。正如我即将发现的，二维码在会议中发挥的作用比我想象的更大。

我在注册处找到了 Li-Wen，扫描了我打印的二维码后，我领到了另一个胸牌和装有赞助商信息的礼品袋。将我的拼音名字从前一晚收到的胸牌复制到这个新胸牌上，只部分成功。某些字符之间的间距有点不对，所以我在下面写上了我的”西方”名字以避免混淆。然后我去了二楼，Li-Wen 已经在那里布置好了我们的 FreeBSD 展台。Philip 已经在那里了，Li-Wen 向我们介绍了他为访客准备的设置。

一个触摸屏显示着两个二维码，参会者可以扫描。第一个是 BSD 台湾 Facebook 群组的链接，第二个是关于 FreeBSD 的小问卷，参与者可以回答。他们向我们展示完成问卷后，我们会扫描他们的个人参与者二维码，他们就能在会议应用中获得一个访问我们展台的印章。收集到足够的印章后，他们可以领取特别奖品。我们周围的其他展台也有类似的安排，从小游戏到调查问卷，让来访展台的人忙碌几分钟。我觉得这是吸引参会者并让他们停留足够长时间以开始对话的绝佳方式。我去过其他会议，感觉人们来展台不是因为兴趣，而是来拿免费赠品，然后很快就消失。通过 COSCUP 的印章游戏，我们更好地了解了受众，因为问卷结果显示大多数人仍是 FreeBSD 新手。此外，如果人们害羞，我们仍然能从他们那里获得反馈。我们与一些过去尝试过 FreeBSD 或在服务器上运行 FreeBSD 多年的人进行了有趣的对话。

大多数参会者之前没有听说过或使用过 FreeBSD。显然，我们需要再来。

COSCUP 让我感兴趣的另一点是工作人员数量之多（都是志愿者）。他们穿着相同的 T 恤，耳朵里带着小按钮和麦克风用于相互沟通。例如，如果冰镇绿茶桶空了（由于高湿度，人们很渴），他们会沟通这一情况，然后有人来送新的一桶。

会议在下午 5 点结束，我们回到酒店，但在此之前我们参观了一家当地的冰淇淋店，那里的选择很有趣，比如罗勒味或猪肘味。Sato-San、Philip 和我几个小时后在一个夜市吃晚饭，这是台北另一个必去的景点。

第二天一早，我们收到消息说会议大楼夜间发生了停电。由于周末无法及时修复，他们将整个会议（演讲、教程、展台）转移到校园内另一栋未受停电影响且仍有电的大楼。一个组织良好的会议的另一个标志可能是组织者和工作人员如何应对这种场景。COSCUP 的工作人员不仅向所有参与者发送了带有新地点地图的邮件，还为当天的演讲和展台找到了新房间。校园里张贴了指示牌，指引我们找到新大楼，就好像这一直都是他们的计划。我从未感到事情因停电而仓促、被遗忘、混乱或缺失。所有组织者做得很好（他们前一晚大概没怎么睡）。

Sato-San 留在展台继续扫描访客的二维码，我去了”BSDTW x Cat System”工作坊听 Philip 关于 ZFS 的演讲。演讲很受欢迎，我留下来听了下一位当地 FreeBSD 维护者关于 Ports 的演讲。我自己的演讲在下午。Li-Wen 为我们两个会议日安排了盒饭，所以我做演讲时并不饿。听众对我演讲的内容感兴趣，我收到了几个问题。一些学生后来找到我，我们在走廊里继续交谈。之后，我回到展台与 Sato-San 一起。

我从会议中观察到的另一点是，女性参会者的比例要高得多。其中一些人在展台或演讲后向我们提问，从她们的问题来看，我可以说她们至少与男性同行在技术知识水平上相当。这是一个非常令人鼓舞的景象。

Sato-San 和我后来接受了”BSDTW x Cat System”工作坊小组成员的采访。她的问题也非常精准，表明她对 BSD 并不陌生。Sato-San 和我就社区参与、如何在这样的会议上更好地推广 BSD，以及我们对改进 COSCUP 的建议等问题做了详尽回答。采访结束时，最后一场会议演讲也结束了，Li-Wen 已经清理了我们的展台并打包了剩余的纪念品。Philip 在演讲后已于那天早上离开了去赶飞机。在走回 MRT 车站的路上，Li-Wen、Sato-San 和我回顾了这次会议，以及下次我们应该在那里提供什么样的演讲或工作坊。我们在附近一家餐厅享用了美味的火锅晚餐，结束了这一天。

总的来说，COSCUP 对我来说是一次有趣的经历。这些年来我去过不同国家的许多会议。COSCUP 给我的印象是组织专业（即使没有停电）。有很多志愿者（看看工作人员列表：<https://coscup.org/2019/en/staffs>）和许多有趣的演讲，其中很大一部分是英文的。

我很高兴去了台北（尽管湿度高且旅行时间长）。人们非常热情地欢迎我（有时出乎意料），与那里的人有很多良好的互动，我觉得这绝对是我们应该更大力推广 BSD 的地方，因为人群对 BSD 操作系统能提供的东西似乎很感兴趣。

---

**Benedict Reuschling** 于 2009 年加入 FreeBSD 项目。2010 年获得完整文档提交权限后，他积极指导其他人成为 FreeBSD 提交者。他于 2015 年加入 FreeBSD 基金会，目前担任副主席。Benedict 拥有计算机科学理学硕士学位，在德国达姆施塔特应用技术大学教授面向软件开发者的 UNIX 课程。他与 Allan Jude 一起主持每周的 BSDNow.tv（<http://BSDNow.tv>）播客。

## vBSDcon

作者：Brad Alexander

vBSDcon 每两年举办一次，我参加了 2015、2017 和 2019 年的 vBSDcon。我在弗吉尼亚州雷斯顿地区工作，离会场凯悦酒店不到一英里，就在会议赞助商 Verisign 对面，隔着杜勒斯收费公路，所以除了 50 英里的回家路程外，我不需要太多旅行——这确实限制了我与其他参会者在夜间逗留的能力。

我仍然认为自己是 FreeBSD 的相对新手，因为我只使用了大约 5 年。作为 Unix 和 Linux 系统管理员约 25 年，我接触过大多数主流 Unix。我最初涉足 FreeBSD 是几件事的巧合。首先，我是 BSD Now 的固定听众，正如 Kris Moore 曾经说的，变得”BSD Curious”。与此同时，Linux 开始实现 systemd。所以，我开始关注 FreeBSD，因为在开源操作系统中，它离 systemd 尽可能远。最后一点，也是让我完全着迷的，是 ZFS，尤其是 ZFS 启动和启动环境。

我最初的项目相对较小。我最初搭建了一台 FreeNAS 来替代我那台没有磁盘冗余的老旧 Linux 文件服务器。后来，我买了一台 HP T610 瘦客户机并安装了 pfSense 来替代我的 Linux 防火墙。所以第一年，我对 BSD 完全是新手，花了很多时间在 ixSystems 和 FreeBSD 基金会展台与 Kris Moore 和 Dru Lavigne 交谈，一点一点地学习。我加入了 PC-BSD、TrueOS、FreeNAS 的 IRC 频道。当时，FreeBSD 给我的印象是，在我工作过的各种 Unix 中，它感觉”恰到好处”。每个商业 Unix 似乎都有自己的特色让它与众不同。FreeBSD 似乎是我经历过的最纯粹的体验之一。

我参加的每一次会议都让我学到了越来越多，因此每次都带着更多自信出现。参会者的友好和他们对新手的包容令人耳目一新，这与我在 Linux 社区多年的经历不同。今年也不例外，我有机会进一步涉足 BSD 的世界。过去一年对我来说的主要变化是，由于 TrueOS 世界的动荡，我最终将日常使用的计算平台——主要是台式机和笔记本电脑——切换到了原生的 FreeBSD 12.0。

这一转变让我能更多地参与社区。在 David Fullard 关于从 FreeNAS 过渡到 FreeBSD 的演讲中，他提到希望 `freebsd-update` 能够感知 ZFS。我写了一个名为 `freebsd-upgrade` 的脚本正好能做到这一点。所以，我把脚本提供给他，我们就这个概念进行了合作。编写 `freebsd-update` 的 Colin Percival 也提出了一些建议。到会议结束时，David 有了一个可用的原型。

我还喜欢 Benedict Reuschling 关于用 PostgreSQL 替代 Oracle 的演讲，以及 Michael W Lucas 关于 jails 的演讲，后者让我更加坚定地将所有 OpenVZ Linux 容器迁移到 BSD Jails 的决心。

第一天晚上，Benedict 在黑客休息室教我基本的 Ansible，他向我展示了如何搭建基本环境。这是我通勤影响我充分利用教学和社交机会的其中一个时刻。

第二天的亮点包括 Allan Jude 关于 ZFS 缓存的”像给 5 岁小孩一样解释给我听”演讲、Colin Percival 关于侧信道攻击的演讲，以及 Conor Beh 关于工作中使用 FreeBSD 的演讲。这让我深有感触，因为 FreeNAS 和 pfSense 是我接触 FreeBSD 的途径。我还与 Mark Felder 谈论了 pcdm 中的一个 bug——在它之后启动的服务无法启动。他帮我在我的系统上修复了它，并正在寻找更永久的修复方案。

总而言之，我非常享受这次会议，感到自己获得了更深更广的知识，并更多地融入了社区。我的目标是明年能参加 BSDCan。

---

**Brad Alexander** 在军队从事卫星通信后转入 IT 行业，在 Unix 系统管理中找到了归宿。多年来，他接触过大多数主流 Unix——SunOS、Solaris、Xenix、AIX、HP-UX、BSDI、Linux 和 FreeBSD。他与结婚 33 年的妻子和他们的狗住在弗吉尼亚州。

## EuroBSDcon 2019 利勒哈默尔

作者：Chris Iroanya

为期四天的 EuroBSDcon 2019 在挪威历史名城利勒哈默尔举行，汇聚了开发者、测试者、用户和爱好者，共同热爱 BSD 及其变体。我是一名网络管理员、FreeBSD 用户，也是 Paul Schenkeveld 旅行资助的获得者——该资助由 EuroBSDcon 基金会提供，以纪念他对 BSD 社区的奉献。我的祖国是尼日利亚，该资助支持了我的旅行、酒店住宿、会议和社交活动。

利勒哈默尔历史悠久、文化丰富，仅穿过这座起伏而宁静的小镇就能感受到。这里举办过两届奥运会——1994 年冬季奥运会和 2016 年冬季青年奥运会——如果挪威赢得了 2022 年冬季奥运会的申办权，还会举办第三届。作为县的行政中心多年，利勒哈默尔对于像我这样热爱美学和文化的人来说也是一个商业上可行的旅游目的地。在挪威乡村绕行两个多小时的火车之旅，是我欣赏该国丰富的畜牧业和美丽风景的绝佳方式。作为游客和会议参会者，我认为 EuroBSDcon 筹备委员会选择利勒哈默尔作为举办地是一个很好的选择。对我来说——我相信对许多其他游客也一样——唯一的缺点是啤酒的价格——是的，我说的是”啤酒”。

会议的前两天（9 月 19 日和 20 日）是教程或开发者峰会。这对我来说是梦想成真，我相信对许多其他人也一样，原因是我不仅与许多 BSD 大牛坐在同一个房间里，还接受他们的指导。另一个原因是我获得了大量有用的知识和信息，还有很多需要后续研究。我有趣的收获之一来自 Kirk McKusick 博士关于”FreeBSD 开源操作系统简介”的讲座，他讲解了 FreeBSD 内核 IPC 子系统中的内存管理、FreeBSD Mbuf 以及使用 FreeBSD 架构的路由设计。

第 3 天和第 4 天更有趣。我喜欢 Patricia Aas 的主题演讲”嵌入式伦理”。她天性中就是我所称的”变形者”，意为能够从她的小世界中推动变革的人。她的演讲可以概括为彼得·帕克原则（蜘蛛侠）：“能力越大，责任越大。“她以此为出发点，表示尽管困难重重，我们有道德责任为 IT 行业的伦理挺身而出。

我 EuroBSDcon 体验的精华是社交活动——我们参观了 Maihaugen 露天博物馆。它占地约 2 平方公里，有 200 栋建筑，是利勒哈默尔游客最多的旅游景点，我们为访客数量添砖加瓦是一件好事。博物馆以保护和展示挪威文化和历史为荣，通过博物馆的三个主要部分来呈现：乡村收藏、历史城镇和住宅区。对于任何访问挪威的人来说，这确实是必看的景点。

会议于 22 日下午约 3:30 闭幕，宣布下一届 EuroBSDcon 将在奥地利维也纳举行。我们大声欢呼，深知奥地利啤酒酿造精良且价格实惠。

总之，我在 EuroBSDcon 2019 的体验非常棒。它给了我很多研究内容，并提供了可探索的项目想法。感谢 FreeBSD 基金会、FreeBSD 社区、会议组织者、演讲者以及每一个让这个项目保持活力的人。期待在奥地利维也纳再次见到你们。在那之前，“Habe ein tolles BSD-Entwicklungsjahr.”——希望我说对了，这句德语的意思是“祝有一个很棒的 BSD 开发之年”。

**Chris Iroanya** 是 Wilforce International Services Limited 的网络管理员，使用 FreeBSD 操作系统搭建网络服务器。他与妻子 Mimi 和两个可爱的女儿 Samantha 和 Chimamanda 住在尼日利亚。他喜欢烹饪和与家人共度时光。
