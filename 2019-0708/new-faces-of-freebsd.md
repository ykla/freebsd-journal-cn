# FreeBSD 新面孔

作者：**DRU LAVIGNE**

本专栏旨在聚焦近期获得提交权限的贡献者，并把他们介绍给 FreeBSD 社区。本期聚焦 3 月获得 src 权限的 Mitchell Horne。

## Mitchell Horne

请介绍一下你自己、背景和兴趣。

- 我在加拿大东海岸长大，目前住在安大略省。我是滑铁卢大学的学生，正在读计算机工程专业的最后一年。我从青少年时期开始折腾 GNU/Linux 发行版，那时就成了计算机爱好者。从那时起，为开源世界做贡献一直是我的目标。

  计算之外，我对喜剧和音乐等表演艺术感兴趣。我做了几年即兴喜剧，也教加拿大高中生即兴喜剧。

你是怎么知道 FreeBSD 的？FreeBSD 哪些方面让你感兴趣？

- 虽然听说过 FreeBSD，但在参加工作之前我从没真正试用过。2018 年初，我有幸被 FreeBSD 基金会录用为实习生。在那四个月里，我接触到了 FreeBSD 的许多不同领域，参与了令人兴奋的项目，比如 **kcov(4)** 的早期版本——这个项目最终让 Syzkaller（<https://github.com/google/syzkaller>）得以在 FreeBSD 上运行。

  从纯 Linux 背景过渡过来一开始有些不适应，但我很快就意识到，FreeBSD 本质上是一个令人兴奋的系统，有很多值得探索的地方。整个操作系统的协调一致、出色的文档，以及不含废话却很友好的社区，这些都让 FreeBSD 在我实习期间成为一件乐事。它也让我看到了还有多少东西要学，这让我想把 FreeBSD 的工作继续做下去，不止于实习期。

你是怎么成为提交者的？

- 实习结束后，我有几个月没提交任何东西——直到 2018 年 9 月左右，我弄到了一台可以专门用于 FreeBSD 开发的机器。从那时起，我开始研究 RISC-V 移植，经过一些初始的搭建困难后，我开始提交小补丁。我发现这是系统中值得投入的领域，因为还有很多工作要做，而积极投入的人不多。此外，这也是学习新兴计算机架构的好机会。

  我持续提交，主要做的功能是为 RISC-V 添加 minidump 支持，这花了我几个月时间。那个项目让我学到很多，因为它触及 FreeBSD 的多个领域和子系统，但幸运的是 Mark Johnston（markj@）和 John Baldwin（jhb@）在 IRC 上通过评审和提问给了很多帮助。提交那个功能后，Mark 问我是否愿意成为提交者，我热情地答应了。

加入 FreeBSD Project 以来你的体验如何？对也想成为 FreeBSD 提交者的读者有什么建议？

- 加入以来我的体验非常积极。社区非常友好，能真正成为其中一员是巨大的荣幸。

  加入以来学业一直很忙，所以我没有像希望的那样活跃。遗憾的是，学业优先，我错过了今年的 BSDCan，但期待明年的大会，到时希望能见到一些我敬仰的提交者和社区成员。

  对想为 FreeBSD 做贡献或成为提交者的人，我的建议是不要气馁。刚开始时我觉得无所适从，因为代码库很大，可以深入的领域太多。如果你还不是某个领域的专家（就像我当时不是），那么我建议短期内学会从小事中获得满足——不管是无关紧要的 bug 修复还是简单的文档改进，都有帮助。相信随着时间推移，你会逐步熟悉系统、积累知识。我读代码的时间仍然远多于写代码的时间，但我总感觉自己学到了新东西。

  此外，我建议尽可能多地寻求帮助。我承认自己在这方面做得不够多，但每当我这样做时，学到的东西比独自摸索要多。这个社区里有很多非常聪明的人，他们乐于与愿意提问的人分享知识。
