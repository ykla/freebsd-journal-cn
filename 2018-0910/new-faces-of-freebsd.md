# FreeBSD 新面孔

作者：**Dru Lavigne**

这个专栏旨在聚焦最近获得提交位的贡献者，并将他们介绍给 FreeBSD 社区。本月的专栏聚焦于 5 月获得 src 位的 Breno Leitao。

跟我们谈谈你自己、你的背景和你的兴趣。

- Breno：我从 1997 年开始使用开源时就对其充满热情。在那之前，我是 MS-DOS/Windows 用户，对自己的机器几乎没有控制权。发现开源后，我开始从不同的角度看待计算机，这让我大开眼界，我第一次真正理解计算机是如何工作的，并第一次感觉到我真正控制了自己的机器。那时，我没有技能在深处改变很多东西，但我有最重要的东西——源代码。

  我开始使用不同的 Linux 发行版和 BBS，我是个快乐的人。大约 1999 年，我拥有一台 486 DX 66MHz，互联网还很新，从互联网下载音乐是你能做的最酷的事情之一。当我终于能够使用 PPP 协议从互联网下载我的第一个 MP3 文件时，我发现我的系统不够快，无法流畅播放 MP3 文件。

  我并没有因为这个发现而沮丧，决定尝试做任何事情来在我唯一能访问的计算机上播放 MP3 文件。在网上阅读资料时，我听说了名为 FreeBSD 的超级快的操作系统。

你是如何第一次了解 FreeBSD 的，FreeBSD 的什么吸引了你？

- Breno：为了能在我的计算机上播放 MP3 文件，我决定清除我的系统并安装 FreeBSD。那时是版本 4，我觉得尝试新事物是个好主意。安装后，我启动了 MP3 解码器，但没有运气，音乐在这个新操作系统上仍然中断。

  那时，我决定深入研究并查看源代码。当然，那时我理解的不多，但我能够理解一些我之前不知道的重要东西。内核是使用主要功能构建的，如果你不使用它们可以禁用。

  我可以重新配置我的内核，避免我不使用的代码，目标是加快 MP3 播放器的速度。

  玩了一段时间后，用不同的编译器标志重新编译了几次，移除多用户能力，我终于能够播放我下载的 MP3。那是我生命中一个伟大的时刻！

  从那天起，我开始玩其他 BSD，并转到 NetBSD 一段时间，因为它是我在 6 张 CD 上随所有软件包一起提供的唯一操作系统。所以，我在大学下载并刻录了 ISO，以便在家拥有完整的存档而无需漫长的下载时间。

  同时，我在大学图书馆找到了《The Design and Implementation of the FreeBSD Operating System》第一版，我感到惊讶。我借了它并开始阅读，但我没有足够的操作系统概念来完全理解它。一遍又一遍地借，直到我买了它，15 年后，我仍在阅读它。

你是如何成为提交者的？

- Breno：离开大学后，我很幸运在 IBM 找到一份做操作系统开发的工作，主要是 Powerpc 上的 Linux。使用 Linux 几年后，我决定在 Power 系统上尝试 FreeBSD。这带回了我小时候的快乐，经过一番折腾，我让 FreeBSD 在 pSeries 平台上运行，BSD 的火焰重新点燃。

  一段时间后，我与 FreeBSD/powerpc 维护者交谈，他们对我愚蠢的问题/代码非常友好和耐心，我终于找到了家的感觉。找到家的感觉对我来说非常重要，能够开始发送补丁。在一个通宵折腾代码的夜晚，我被 Justin Hibbits 和 Nathan Whitehorn 邀请成为提交者。在那一天之前，他们对我非常耐心，我想借此机会说声“谢谢”。

自从加入 FreeBSD 项目以来，你的经历如何？你对可能成为 FreeBSD 提交者的读者有什么建议？

- Breno：这种经历是独特的，主要是因为开发者只在一条 IRC 消息之遥，这让我着迷。我可以 ping 编写任何特定代码的人并问（主要是）愚蠢的问题，他们非常合作，到目前为止非常有帮助。高级开发者平易近人且乐于支持，对我来说是一种罕见的经历，因此，这个项目的独特性。

  我不太擅长给人建议，但我用问题纠缠 Powerpc 维护者（有时整天）这一点，对我来说是能够解决重要问题并被邀请成为提交者的关键。

**DRU LAVIGNE** 是 iXsystems 的存储工程总监，《BSD Hacks》和《The Best of FreeBSD Basics》的作者。
