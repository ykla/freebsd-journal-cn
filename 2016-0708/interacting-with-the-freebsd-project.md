# 与 FreeBSD 项目互动

基金会聚焦

- 原文：[Interacting with the FreeBSD Project](https://freebsdfoundation.org/wp-content/uploads/2016/08/Interacting-with-the-FreeBSD-Project.pdf)
- 作者：**Deb Goodkin**

我很喜欢住在科罗拉多州博尔德。这里位于落基山脉脚下，几乎每天（好吧，几乎是每天）都有很多机会去户外活动。我每天通常花很长时间运营基金会、支持项目，但总会抽时间跑长跑或骑自行车。这是离开电脑、好好锻炼一番、在脑海里整理基金会工作的好机会。

有时，出门走走能让我以更宏观的视角看待工作。最近一次跑步时，我正设法让自己的大脑从炎热中分心，开始回想从 FreeBSD 社区朋友那里收到的一些邮件。我们收到许多向我们求助的邮件——可能是请求软件开发工作，也可能是请我们为项目处理某些事务。其中一些信息让我意识到，大家对我们规模大小、能力范围存在误解。

我想借此机会向你介绍我们是谁、在做什么，重点介绍近期帮助 FreeBSD 的活动，并谈谈我们未来的方向。

## 我们的起源

1999 年，Walnut Creek CDROM 想把 FreeBSD 商标所有权转交给 FreeBSD 项目。然而，项目由若干个人组成，并非法律实体。当时在核心团队的 Justin Gibbs 开始研究如何合法持有商标。他发现，成立一家非营利组织不仅能持有商标，还能持有并保护所有 FreeBSD 知识产权。于是他在 2001 年初创立了 FreeBSD 基金会。

最初五年，我们仅靠维持董事会所需的最少志愿者运营——三个人。资金有限、大家都有本职工作，所以我们专注于少数几个定向资助项目，支持新兴的 BSD 会议，并与 Sun 等公司签订许可协议，提供 FreeBSD Java 二进制文件。

随着时间推移，Justin 意识到当时的状况无法持续，2005 年他聘请我运营这家机构。我们扩充了董事会，我担任董事会财务长/秘书。我们都是董事会成员，所以别人想联系我们时，只需发邮件给 **<board@freebsdfoundation.org>**（注：此邮箱现已停用！）。基金会和董事会基本是一回事。

## 基金会成长！

时间快进到 2016 年。如今我们有六名工作人员和七名志愿者董事会成员。董事会帮助制定战略方向、与各方互动、筹集资金，并承担信托责任。工作人员则负责落实战略计划、积极运营机构，并支持项目。

Sabine Percarpio 2016 年加入基金会，担任行政经理。她处理捐赠流程、监督会计、管理办公室、协助人力资源和员工福利、管理旅行资助，并且很可能是第一个接听你打给基金会的电话或回复你邮件的人。

这就是我们的团队！我们热衷于支持 FreeBSD，但人手如此之少，能提供的支持有限。我们收到的请求（例如请我们填补项目中的更多空缺）表明需要扩大规模以支持这些工作。然而，只有获得捐赠支持这种增长，才可能实现。

## 我们为 FreeBSD 做了什么？

我们的使命是支持 FreeBSD 项目和全球社区。可以想见，这涵盖很多领域。因此，我们根据现有资金，聚焦能提供的支持，既要壮大社区，又要持续让 FreeBSD 成为稳定、安全、创新且可靠的操作系统。以下是过去几个月我们为帮助 FreeBSD 所做的一些工作亮点。

### 筹款工作

首先我想指出，我们的工作 100% 由你的捐赠资助。截至 6 月 30 日，我们已筹集 257,570 美元，支出 459,900 美元，2016 年筹款目标为 1,250,000 美元。我们的第一、二季度财务报告将于 8 月初发布，届时你可以看到我们如何花钱。要继续维持这种支持力度，我们需要你的捐赠。请考虑在此捐赠。

### 操作系统改进

基金会通过资助经提案流程批准的软件开发项目以及内部软件开发员工来改进 FreeBSD。我们最近完成了一个项目，提升 vnet 网络栈虚拟化基础设施的稳定性。另一个仍在进行的项目，为 FreeBSD 的 64 位 ARM 移植带来新功能以及稳定性和性能增强。我们还在赞助将 blacklistd 引入 FreeBSD 的工作，这是一种基于连接的安全控制的实用方法。

此外，我们的软件开发者持续进行大量改进、实现新功能并修复 bug，保持 FreeBSD 的可靠和稳定。以下是过去几个月完成的部分工作列表：

- 实现稳健互斥锁支持，这是让线程库符合 POSIX 并功能完备的持续工作的一部分。
- 为线程库使用的内核接口编写文档，产出近 30 页技术文本。
- 完成并提交从 amd64 pmap 中移除 `pvh_global_lock` 的工作，消除了一个竞争激烈锁的瓶颈。
- 新增文件系统吞吐量资源控制限制（Rctl）。
- 提交 iSER initiator 支持。
- 新增支持重新挂载到 NFS。
- 新增 `iscsictl`，可启用和禁用 iSCSI 会话。
- 调查 Ports 树中可重现构建的现状，正在解决所发现的问题。
- 更新 ELF Tool Chain，修复 bug 并改进对格式异常输入的处理。
- 调查使用 LLVM 家族的链接器 `lld` 链接 FreeBSD 基本系统，调查并测试所发现问题的修复。
- 管理 arm64 开发项目，调查并修复多个 bug。
- 导入 llvm libunwind 并为在 FreeBSD 中使用做准备。
- 调查并审查 blacklistd 提案和补丁。
- 为网络传输、DTrace 和图形栈组织双周电话会议，帮助协调这些领域的工作。

### 发布工程

如前所述，基金会雇有一名全职人员 Glen Barber 从事发布工程，确保发布可靠、稳定且按时。上一季度，Glen 协助了 10.3-RELEASE 并启动了 11.0-RELEASE 周期。

### FreeBSD 入门项目

我们启动了新的 FreeBSD 入门项目。该项目旨在为首次尝试 FreeBSD 的新人开发 how-to 指南、入门指南和视频。在活动中推广 FreeBSD 时，我们发现许多人对尝试 FreeBSD 感兴趣，因此作为教育倡议的一部分，我们决定提供易于跟随的教程，让这些人快速上手！

在这个项目中，我们引入了一位实习生，他没有 FreeBSD、Linux 或任何命令行操作系统的使用经验，让他自行摸索如何安装和使用 FreeBSD。我们这样做是为了从新人的视角了解使用现有文档的体验，以及哪些因素让学习和在桌面安装 FreeBSD 变得容易或困难。在社区成员偶尔的帮助下，他正在撰写易于跟随的 how-to 指南，让新用户更容易上手。这些指南也很适合在编程马拉松中帮助初学者入门 FreeBSD。

你可以在这里查看我们的 how-to 指南。

### FreeBSD 宣传与教育

我们很大一部分精力用于为项目做宣传。我们通过多种方式做到这一点，包括推广他人使用 FreeBSD 所做的工作，制作宣传文献教授人们 FreeBSD，同时帮助简化使用 FreeBSD 或为项目做贡献的路径。我们还参加 FreeBSD 与非 FreeBSD 活动，并与其他 FreeBSD 贡献者合作，作为志愿者运营 FreeBSD 活动、值守 FreeBSD 展台并发表 FreeBSD 演讲。

我们近期的一些成果包括：

- 在我们网站上创建 FreeBSD 页面，推广 FreeBSD 衍生项目并展示 FreeBSD 用户。
- 通过在我们的网站和会议传单上创建研究页面，推广 FreeBSD 相关研究。
- 创建使用项目和基金会徽标的指南和代码仓库。
- 通过发布两篇关于 Michael Lucas 和 Kris Moore 的新《FreeBSD 之面孔》故事，展示 FreeBSD 贡献者。
- 我们出版这份杂志，这是提供 FreeBSD 信息丰富且有趣文章的又一个平台。
- 在 FreeBSD 教育方面，George Neville-Neil 和 Robert Watson 继续在 teachbsd.org 教授和开发开源 FreeBSD 教学材料。

### 微软喜爱 FreeBSD！

过去两年我们与微软合作，让 FreeBSD 进入 Azure Marketplace。微软将维护自己的 FreeBSD 镜像，并向其客户群提供 FreeBSD 支持。这是 FreeBSD 项目与微软之间令人兴奋的合作。你可以在此处阅读更多关于 Azure 上 FreeBSD 的信息。

### 会议与活动

FreeBSD 基金会在全球赞助许多会议、活动和峰会。这些活动可以是 BSD 相关、开源，或面向代表性不足群体的技术活动。

我们支持 FreeBSD 专题活动，为分享知识、共同开发项目、促进开发者与商业用户之间的合作提供场所。这一切都有助于营造健康的开源生态。我们支持非 FreeBSD 活动是为了推广 FreeBSD、提升其知名度，增加 FreeBSD 在不同应用中的使用，并为项目招募更多贡献者。最后，我们向 FreeBSD 贡献者提供旅行资助，让他们有机会面对面交流知识、合作项目，并通过参加演讲、演示和教程更多地了解 FreeBSD。
