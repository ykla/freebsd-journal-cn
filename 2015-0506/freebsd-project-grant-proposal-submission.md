# FreeBSD 项目资助申请提交

- 原文：[FreeBSD Project Grant Proposal Submission](https://freebsdfoundation.org/our-work/journal/browser-based-edition/measure-twice-code-once/)
- 作者：**Ed Maste**

FreeBSD 基金会是一家总部位于美国的非营利组织，致力于在全球范围内支持并推广 FreeBSD 项目与社区。基金会通过多种方式履行这一使命，包括赞助 FreeBSD 活动与差旅、提供法律支持、资助和管理开发项目。开发资助用于支持特定项目或操作系统特性的开发。

本文中我将讨论提交资助申请的流程及其应包含的细节。资助申请针对具体、可操作的项目，并需指明完成工作的开发者、成本信息。FreeBSD 基金会一直希望了解 FreeBSD 社区的愿望与目标，并希望收到关于项目开发列表优先级排序的反馈。不过那种反馈是在资助流程之外接收的。

基金会会不时发布明确的提案征集。这样可以把申请放在一起对比审议，使决策过程更高效、更及时。提案征集可能会包含由基金会总体目标所指引的特定关注领域信息。尽管我们更希望在提案征集的背景下收到申请，但我们也随时接受主动提交的申请。

资助申请可由个人开发者或公司提交。申请可涉及 FreeBSD 操作系统或项目内任何子系统或基础架构的工作。新的内核驱动、用户态库与工具、第三方软件移植、性能优化、网络栈研究、文档、基于 Web 的项目工具，都是基金会会考虑的项目示例。

FreeBSD 社区中的任何人都可提交申请——申请并不局限于 FreeBSD 项目的提交者。然而，非提交者提交的申请通常需要指明 FreeBSD 项目中的某人，由其负责将工作引入 FreeBSD。

项目规模在时间或成本上没有固定限制。一份申请可以描述只需几天的工作，也可以是几个月的工作。不过，针对关注领域有一些通用指引。我们尤其鼓励针对软件、文档或基础设施的申请，这些领域在 FreeBSD 社区中服务不足。也就是说，基金会特别关注那些填补志愿者工作或他人已资助工作空白的申请。

所提议的项目应与基金会推广的其他项目、目标与技术相整合。例如，新的守护进程和用户态工具应考虑使用 Capsicum 能力与沙箱框架。软件开发项目必须包含相关文档产出的说明——新增或更新文档是一项基本要求。还应提及测试，无论是单元测试、系统测试还是其他形式，并且新测试通常应与 FreeBSD 的 ATF/Kyua 测试框架整合。

最后，请考虑为所提项目做宣传；基金会的目标之一是让更广泛的开源与自由软件社区了解 FreeBSD 取得的进展。这可以通过博客文章、BSD 专题或其他大会上的演讲，或针对成功项目的杂志文章来实现。

## 撰写资助申请

资助申请按技术价值、成本效益、项目对 FreeBSD 的影响程度来评估。成功的申请应清楚说明项目将带来的收益。基金会网站有提交所需格式的完整说明，此处给出一些指引。

### 标题

为申请选一个简短但好记的标题。基金会会在公告、博客文章和通讯中引用该项目。在提交项目时就选定一个好标题，有助于在沟通中保持一致。

### 联系信息

务必包含你的姓名、（如适用）公司名称、电子邮件与邮政地址、电话号码。

### 摘要

摘要或概述用 50 到 100 字简要介绍整个项目。应描述高层目标、收益、将采取的方法。基金会可能在通讯或其他出版物中使用该摘要。

### 引言

较长的申请、细节复杂的申请应有一个引言，提供项目必要性的历史或背景信息。简短简单的申请可省略。

### 项目描述

描述应充分说明要做什么、为什么需要做、为什么对 FreeBSD 项目和社区有价值。应表明提交者具备完成该项目的资格，并拥有在所述时间和约束内完成项目所需的资源。描述还应包含项目交付物、开发流程概述、测试与文档相关内容。

### 技术监督

技术监督负责跟踪项目进展并确保项目产出质量达标。监督人通常是来自 FreeBSD 社区的志愿者或基金会员工，特殊情况下也可能是付费专家。

### 项目成本

申请必须包含完整、详尽的成本信息，包括开发者费用、任何所需的硬件采购、评审人报酬、适用的税费和附加费用。如果项目有可选组件，可用单独的小计标注，但即使没有任何可选组件获得资助，基础项目也必须可行。

基金会对项目总成本或时薪没有固定指引，而是逐份评估申请的优劣。资助用于支持提交者希望承担以改进 FreeBSD 项目的工作。因此，所提成本通常不可与专有或商业咨询费率相提并论。

### 时间线与里程碑

申请应包含一张表，列出提议的启动日期、每个里程碑或交付物的预期完成日期。如果项目包含分期付款，则需列出每个里程碑的成本。

### 简历

申请应包含两到三段，描述提交者的经验及其在 FreeBSD 项目中的经历或关系。

## 提交流程

基金会网站说明了申请提交流程（<https://www.freebsdfoundation.org/documents/#Proposals>）。完整的《Proposal Submission Guidelines》也可在该处获取，其中包含一份示例申请（<https://www.freebsdfoundation.org/documents/FreeBSDProposalSubmission.pdf>）。

可以提交草案申请以获得初步评审和意见，帮助完善最终版本。草案申请必须明确标注为草案。

最终申请准备好后，应作为文本或 PDF 文件通过电子邮件发送至 <proposals@freebsdfoundation.org>。FreeBSD 基金会项目委员会将评审申请并回复决定。

申请获批后即可开始项目工作。项目进行期间需每周提供简短状态更新，并需在完成每个里程碑时附上更详细的阶段报告和最终报告。

我们期待支持你改进 FreeBSD 项目！

## FreeBSD 基金会项目资助示例

- 将 FreeBSD 移植到 AArch64，即 64 位 ARMv8 架构
- PCIe 热插拔基础设施
- 用现代 AES 模式更新 opencrypto
- 更新系统视频控制台
- FreeBSD 的 Multipath TCP（硕士研究项目）
- Capsicum 应用沙箱框架集成
- ARMv7 的超级页支持
- 新的 TCP 拥塞控制算法
- FreeBSD IPv6 协议栈性能分析
- 在已挂载文件系统上支持 growfs
- 文档项目基础设施增强
