# 会议报告：BSDTW 2017

作者：**BROOKS DAVIS**

2017 年 11 月 11 日和 12 日在台湾台北举办的 BSDTW 2017 取得了圆满成功。我以演讲者和 BSD 开发者的双重身份参加。这是首届 BSDTW，也是台湾第二次举办 BSD 大会。会议由 Open Culture Foundation 和 Skymizer 主办，Li-Wen Hsu 牵头组织。

会议规模比许多会议小，只有一个演讲轨道，避免了我想听的演讲挤在同一时段的常见问题。开幕式后，第一天以 Arun Thomas 关于 RISC-V 架构的演讲开场。Arun 向我们介绍了该架构的概貌、当前状态更新、即将到来的变化预览。RISC-V 似乎在多个领域（包括工业和研究）日益受到关注，FreeBSD 对其提供扎实支持是件好事。

接下来是 George Neville-Neil 关于 CADETS 项目中使用 DTrace 的更新。他们以常驻模式使用 DTrace，而传统 DTrace 的性能保证（探针未启用时影响极小或无影响）并不适用。George 介绍了多项正在进行的改进和 OpenDTrace 的工作——这是一个旨在沿 OpenZFS 的思路将各 DTrace 社区重新聚拢的项目。

随后 Baptiste Daroussin 讨论了扩展 Poudrière 包构建系统以构建 FreeBSD 镜像的工作。从某种角度看，这似乎偏离了 Poudrière 的初衷，但实际上 Poudrière 一直需要构建干净的 FreeBSD 镜像，以便在构建包时挂载为 OS 文件系统。新功能只是增加了打包这些镜像的能力，而非继续作为 ZFS 卷维持。午饭后，我做了关于虚拟地址空间管理历史的演讲，展示了我们从早期 UNIX 的 `break()` 接口如何演进到今天的 `mmap()`。我还讨论了为什么我认为面对现代缓解策略和 API 设计理念，是时候重新审视这个接口了。

Johannes M. Dieterich 随后做了关于 FreeBSD 在 HPC 领域现状的演讲，聚焦程序语言支持、数值库和 GPU 支持。作为曾在该领域工作过的人，看到 FreeBSD 的进展和哪些地方仍需努力，很受启发。

当天最后一场演讲由 Peter Grehan 带来，他讨论了为 bhyve 添加图形支持的挑战。这项工作的动机在于：没有图形支持，就没有实际可行的方式安装非服务器版 Windows 和某些 Linux 发行版。bhyve 现在是模拟的图形适配器、键盘和鼠标加上 VNC 服务器。这场演讲充满了有趣的示例，展示了看似标准的东西（如 VNC 协议）在实践中往往不那么标准。

当晚，会议晚宴在淡水区一家餐厅举行。我们从捷运线终点站沿海滩走到餐厅，沿途有许多卖食物的店铺，包括一些看起来很棒的炸头足类海鲜串。可惜我们赶着去吃晚饭，没时间品尝。

周日上午由 Allan Jude 开场，讲的是高级 ZFS 集成。他的演讲主要聚焦于启动过程的各种增强，包括 boot environment、新的 ZFS 下次启动实现、自动启动恢复。Allan 还谈到了 GELI 磁盘加密和使用 ZFS 升级设备。

接着是 Theo de Raadt，带来了一场关于使用安全缓解措施逐步提升安全、及早发现 bug 或代码质量问题并修复的精彩演讲。他很好地论证了 OpenBSD 采取的默认开启方式。这种方式并非适合所有人，但有人这样做十分难得。

Ruslan Bukin 做了关于 FreeBSD 和更广泛软件生态中 RISC-V 支持现状的演讲。他还带我们了解了移植过程，并讨论了为支持即将到来的 v1.10 特权规范所做的改动。树内 Linux 支持将基于此版本，因此应接近最终形态。

午饭后，Mariusz Zaborski 讨论了用 Capsicum 沙箱化应用。除其他内容外，他详述了各项服务支持现有 libc 接口的原因。一个新的服务（演讲前后提交）是 syslog 服务，之所以需要它，是因为当前 API 中没有可靠方式预打开日志服务。

会议最后一场完整演讲由 Mark Johnston 带来，讲的是 FreeBSD 虚拟内存系统的变化。Mark 涵盖了广泛的改进，从近期为防止某些 stack clash 问题而加入的 `MAP_GUARD`，到 128 线程系统上构建性能的提升——这些系统现在 Amazon 上可用。

会议以闪电演讲收尾，时间限制严格执行。演讲内容有趣，计时员让演讲者保持紧凑。

我抽空做了一些旅游活动，漫步台北街头，并在一个晴天登上了台北 101。可惜行程中空闲时间不够，没能参观更多有趣的地方，比如故宫博物院。我期待未来的 BSDTW 大会，下次到台湾时去看看这次错过的地方！
